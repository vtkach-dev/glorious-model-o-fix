# glorious-model-o-fix
Проблема: не запускается софт для мышки (ни glorious model o software, ни glorious core)
Решение (Windows 10 22H2):
Качаем glorious model o software с touslesdrivers.com или официального сайта (именно легаси версию) (может потребоваться VPN). Также можете скачать с этого репозитория.
Заходим в пуск -> диспетчер устройств -> Мыши и иные указывающие устройства. Ищем свою мышь по дате установки / пробуем удалить устройство - смотрим, что отключилось.
Когда нашли, заходим в свойства -> сведения, выбираем в выпадающем списке "ИД Оборудования".
Список будет выглядеть примерно так:

HID\VID_258A&PID_0033&REV_0108&MI_00
HID\VID_258A&PID_0033&MI_00
HID\VID_258A&UP:0001_U:0002
HID_DEVICE_SYSTEM_MOUSE
HID_DEVICE_UP:0001_U:0002
HID_DEVICE

Нам нужна только первая строчка.
VID устройства после VID_ (в данном случае 258A)
PID устройства после PID_ (в данном случае 0033)
Мне также пришлось взять еще MI_ (тут 00)

Распаковываем .zip софта, в нем две папки (01 FIRMWARE UPDATE и 02 SOFTWARE)
Заходим в 02 SOFTWARE и запускаем установщик. ОБЯЗАТЕЛЬНО ВБИРАЕМ ГАЛОЧКУ С "Запустить программу / Launch program" (в таком роде)
Идем по пути, куда установилось (по умолчанию в Program Files x86 -> Glorious model O software)
В папке один .ini конфиг, не пропустите
Открываем в редакторе от имени администратора (иначе не даст сохранить)
Ищем строку, где написано [MS]
Вставляем свои VID и PID в формате 0xVID, 0xPID
Я дополнительно пролистал до строчки PSD и дописал в конец свой MI_ (0x00)
Сохраняем конфиг
Запускаем OemDrv.exe, он выдаст сообщение об ошибке PSD (у меня выдал по крайней мере), игнорируем и ждем, софт должен запуститься.
Получаем рабочий с костылями софт!

ЕСЛИ НЕ РАБОТАЕТ

Проверяем VID, возможна мышка поддельная
Ну или не покупаем флагманскую в прошлом мышку за 500р б/у
